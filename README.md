# scanitycheck-crennotech
Internal vulnerability scanning platform for Crennotech : built with OWASP ZAP, Nuclei, .NET 8 backend and React frontend. Developed as part of University Industry Capstone Project.

## Project Overview
ScanityCheck is an internal vulnerability scanning tool that scans web applications and APIs for security vulnerabilities using OWASP ZAP and Nuclei, with results displayed on a React dashboard.

## Tech Stack
- **Backend** : .NET 8 Web API
- **Frontend** : React + Vite + shadcn/ui
- **Database** : SQL Server
- **Scanning Tools** : OWASP ZAP, Nuclei
- **Background Jobs** : Hangfire
- **Cloud** : Microsoft Azure

## Folder Structure
**/backend**      → .NET 8 Web API, scan orchestrator, Hangfire jobs\
**/frontend**     → React dashboard, scan submission, results display\
**/docs**         → Sprint deliverables, diagrams, design documents

## Docker Setup Instructions
The easiest way to run the entire stack (React frontend, .NET 8 API, and SQL Server) is with Docker Compose. The application uses environment variables for security and configuration. You must set these up before starting the containers.

1.  **Clone the repository**

2.  **Configure variables:** Copy the `.env.example` file to create a local `.env` file in the project root directory
    * Fill in the `[REQUIRED]` values. Detailed instructions and requirements for each variable are documented directly within the `.env.example` file.

3. **Build and start:** `docker-compose up --build`

4. **Stop the application:** `docker-compose down`

#### Notes
* The API service is configured to wait for the SQL Server to be "Healthy." You can monitor the startup status using: `docker-compose ps`
* Environment for the API service is configured via `ASPNETCORE_ENVIRONMENT` in `docker-compose.yaml`
* Completely reset the environment (including deleting database): `docker-compose down -v`

### Service Endpoints

Once the containers are running, the services are exposed at the following URLs:

| Service      | URL                                         |
| :----------- | :------------------------------------------ |
| **Frontend** | [http://localhost:5173](http://localhost:5173)   |
| **Backend API (base URL)** | [http://localhost:8080](http://localhost:8080) |

**Note**: The backend does not expose a response at `/`. API documentation (Swagger UI) is available at `/swagger`.

### Database Details

Use the following details to connect to the database:

| Field      | Value                                         |
| :----------- | :------------------------------------------ |
| **Server** | `localhost,1433`   |
| **User** | `sa` |
| **Password** | `MSSQL_SA_PASSWORD` (set in `.env`) |
| **Database** | `ScanityCheckDb` |

---

### Project Structure & Volumes

- **Database Persistence**: The SQL Server uses a named volume `mssql_data` so your data persists even if containers are stopped
- **Reports**: Security scan reports generated by the API are synchronised to the local `/reports` folder in the project root directory
- **Hot Reload**: The API is volume-mapped to the source code to support a smoother development workflow
